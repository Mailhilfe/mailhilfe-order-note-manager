# Support

Use GitHub issues for reproducible bugs and feature requests that do not contain sensitive information.

A useful bug report includes:

- plugin version
- WordPress version
- WooCommerce version
- PHP version
- HPOS status
- current locale
- exact reproduction steps
- expected and actual behavior
- relevant `WP_DEBUG` messages with private data removed

For security issues, follow `SECURITY.md` instead of creating a public issue.
