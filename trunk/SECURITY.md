# Security Policy

## Supported versions

Security fixes are applied to the latest stable release line.

| Version | Supported |
|---|---|
| 2.0.x | Yes |
| Older versions | No |

## Reporting a vulnerability

Do not open a public GitHub issue for a suspected security vulnerability.

Send a private report to:

```text
info@mailhilfe.de
```

Include:

- affected plugin version
- WordPress and WooCommerce versions
- whether HPOS is enabled
- clear reproduction steps
- impact assessment
- proof of concept, if available

Do not include real customer data, credentials, API keys or live order information.

## Coordinated disclosure

Please allow reasonable time to investigate and prepare a fix before public disclosure. Acknowledgement and release timing depend on severity and reproducibility.
